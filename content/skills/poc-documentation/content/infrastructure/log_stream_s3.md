## Overview

Aembit retains the last 24 hours of activity in the Management Console reporting tab. Configuring an S3 Log Stream exports Audit Logs, Access Authorization Events, and Workload Events to your own S3 bucket for long-term retention and SIEM ingestion.

## Prerequisites

- AWS S3 bucket already created in `{{AWS_REGION}}`
- Permission to modify the S3 bucket policy
- Aembit tenant with Administrator access

## Values Reference

| Value | Where to Find It |
|-------|-----------------|
| `{{AWS_ACCOUNT_ID}}` | AWS Console → top-right account menu, or: `aws sts get-caller-identity --query Account --output text` |
| `{{AWS_REGION}}` | AWS region where the S3 bucket resides (e.g., `us-east-1`) |
| `{{S3_BUCKET_NAME}}` | Name of the target S3 bucket |
| `{{S3_PATH_PREFIX}}` | Optional folder prefix within the bucket (e.g., `aembit/logs/`) — leave blank to write to the bucket root |

## Service Configuration

### AWS S3 Setup

1. In the Aembit Management Console, navigate to **Administration → Log Streams** and click **+ New**

2. Note the **Destination Bucket Policy** that Aembit generates in the dialog — you will need to copy this before saving

3. In the AWS Console, navigate to the S3 bucket `{{S3_BUCKET_NAME}}`
   - Open **Permissions → Bucket policy**
   - Paste the bucket policy copied from Aembit
   - Click **Save changes**

## Aembit Configuration

1. Return to the Aembit **Log Streams** dialog and complete the fields:
   - **Name**: descriptive label (e.g., `S3 Audit Logs`)
   - **Description**: optional
   - **Event Type**: select the event type to export (**Audit Logs**, **Access Authorization Events**, or **Workload Events**)
   - **Destination Type**: select **AWS S3 using Bucket Policy**
   - **S3 Bucket Region**: `{{AWS_REGION}}`
   - **S3 Bucket Name**: `{{S3_BUCKET_NAME}}`
   - **S3 Path Prefix**: `{{S3_PATH_PREFIX}}` (optional)

2. Click **Save**

3. Repeat step 1 for each additional event type (each event type requires a separate Log Stream entry)

## Verification

- Each Log Stream entry shows as **Active** in the **Administration → Log Streams** list
- Perform a test action in the Aembit Management Console (e.g., view an Access Policy) to generate an Audit Log event
- In the AWS Console, confirm new log files appear in `s3://{{S3_BUCKET_NAME}}/{{S3_PATH_PREFIX}}` within a few minutes
- If using a SIEM (e.g., Datadog, Splunk), confirm events are ingested from the S3 bucket

## Troubleshooting

- **Log Stream shows Inactive or error status:** The S3 bucket policy was not applied or was applied incorrectly. Re-copy the policy from the Aembit Log Stream dialog and re-apply it to the bucket under **Permissions → Bucket policy**
- **No files appear in S3:** Verify the **S3 Bucket Region** matches the actual bucket region. A region mismatch causes silent write failures
- **Access denied writing to S3:** The bucket policy does not grant write access to the Aembit principal. Delete the existing bucket policy and re-paste the policy generated by Aembit. Do not modify it manually.
- **Events missing from SIEM:** Each event type (Audit, Authorization, Workload) requires its own Log Stream entry. Confirm a Log Stream exists for the event type you expect to see
