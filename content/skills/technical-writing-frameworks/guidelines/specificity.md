# Specificity Guidelines

Every recommendation must answer three questions with concrete, actionable details.

## The Three Questions

### 1. WHAT needs to change?
- Concrete deliverable (not abstract concept)
- Quantified when possible (how many, how long)
- Clear format (code example, diagram, section, link)
- Specific content focus (not just "more detail")

### 2. WHERE should it change?
- Section name or heading
- Relative position (after X, before Y, replace Z)
- Line numbers or landmarks when available
- Clear enough for someone unfamiliar with the doc

### 3. WHY does it matter?
- Impact on specific personas
- Connection to documented feedback
- Framework justification (7-Action or Diataxis)
- Consequence of NOT fixing

## Templates by Recommendation Type

### Add New Content
What to add (detailed list, format, scope) + Where (section, position, context) + Why (personas, blocker, framework) + Example of content

### Restructure Content
Current structure + Proposed structure + Specific moves (move X from here to there) + Why (user flow, framework alignment, personas)

### Remove/Condense Content
What to remove (section, location) + Rationale (action mismatch, audience mismatch, redundancy, framework violation) + Where to put removed content (delete, move, link) + Wayfinding to add

### Fix Clarity/Accuracy
Current problem (location, issue, evidence) + Proposed fix (current text, replacement text) + Why (accuracy, clarity, terminology)

### Add Wayfinding
Where to add (section, position) + Format (callout, inline link, "Next Steps" section) + Exact text + Links to (document, section) + Why (persona affected, action mismatch, current vs desired behavior)

## Anti-Patterns

| Anti-Pattern | Fix |
|-------------|-----|
| "Improve the tutorial" | "Add 5 concrete steps with code examples to the Setup section" |
| "Add examples throughout" | "Add examples in sections X, Y, Z [specific positions]" |
| "Make it clearer" | "Define terms [list] on first use, add glossary link" |
| "Add troubleshooting section" | "Add troubleshooting because 3 personas reported being blocked by errors" |
| "Better align with user needs" | "Restructure to address Use action instead of Onboard based on Diataxis How-To principles" |

## The Handoff Test

Could you hand this recommendation to someone who has never seen this document, and they could implement it without asking clarifying questions?

- WHAT is clearly defined with concrete deliverable
- WHERE is precisely located with landmarks
- WHY is justified with persona impact and framework backing
- Example or before/after text provided when applicable
