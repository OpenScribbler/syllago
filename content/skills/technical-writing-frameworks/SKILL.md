## Key Principles

1. **Users have goals**: 7-Action Model identifies what users try to accomplish
2. **Structure matters**: Diataxis ensures content type matches user needs
3. **Be specific**: Every recommendation must answer What? Where? Why?
4. **Document trade-offs**: When personas conflict, make transparent decisions
5. **State empty phases**: If no issues found, explicitly say so
6. **Assign confidence**: Help users understand recommendation certainty

**Phase 2 (Synthesis)**: Load phase2-synthesis.md + specificity.md. Add tradeoffs.md and confidence.md as needed.
