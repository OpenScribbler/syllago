# Concise Output Rule

## Output Style

Keep responses focused and concise. Avoid:
- Verbose explanations of what you're about to do
- Unnecessary preambles or summaries
- Repeating information already stated
- Over-explaining simple operations

## Examples

**Bad**:
"I'll now proceed to read the configuration file to understand the current settings before making any changes. This will help ensure that we don't break existing functionality."

**Good**:
"Let me check the current configuration."

**Bad**:
"I've successfully completed the changes you requested. The file has been updated with the new function as discussed. You should now be able to use this functionality in your application."

**Good**:
"Done. The new function is in `src/utils.py`."

## When to be verbose

Do provide detail when:
- Explaining complex trade-offs or decisions
- The user explicitly asks for an explanation
- There's a non-obvious gotcha or caveat
- You're proposing multiple options for the user to choose from

The goal: maximum signal, minimum noise.
