## v0.7.0-beta.1

> First public preview of the 0.7.0 cycle. Cut to exercise the end-to-end
> release pipeline and power the Syllago Docs site against a real, taggable
> build. Expect further changes before the final 0.7.0. Pin to specific beta
> tags in anything you don't want to break.

### Highlights

**TUI v3 and 7-Verb CLI** — Complete TUI rewrite. Card grid pages for Library, Loadouts, and Registries. Full mouse support (click-to-select, click-away modals, scroll wheel), toast notifications, breadcrumbs, and a `?` help overlay on every screen. CLI verbs are now `add`, `remove`, `install`, `uninstall`, `share`, `publish`, `sync`, and `convert`. The removed `import`, `export`, and `promote` commands print guided redirects.

**Canonical Keys Expansion** — `docs/spec/canonical-keys.yaml` now covers all six content types (skills, rules, hooks, mcp, agents, commands) with an explicit confidence model, key lifecycle policy, and public-spec inheritance rules. Provider `canonical_mappings` are populated across every supported content type, making cross-provider conversion deterministic end-to-end.

**Capabilities Pipeline** — New `_gencapabilities` command plus a full `capmon` subcommand suite (`check`, `derive`, `onboard`, `validate-format-doc`, `validate-sources`) that keeps provider format docs, capability YAML, and canonical mappings in sync. `docs/provider-formats/` now holds structured format docs for 14 providers with rules/hooks/mcp/agents/commands sections.

### New Features

#### CLI Commands
- **`syllago add`** — Discovery mode with positional args, item-level discovery, and interactive provider detection flow. Replaces `import` for the common cases.
- **`syllago remove`** — Remove content from project or global library.
- **`syllago install` / `syllago uninstall`** — Install or uninstall content to/from specific providers.
- **`syllago share`** — Promote project content to the global library.
- **`syllago convert`** — Convert content between providers through syllago's canonical format.
- **`syllago capmon`** — Capability-monitor subcommands: `check`, `derive`, `onboard`, `validate-format-doc`, `validate-sources`.
- **`syllago _gencapabilities`** — Internal command that regenerates per-provider capability YAML from authoritative format docs.

#### Global Content Library
- **`~/.syllago/content/`** — Content shared across projects. Global items show a `[GLOBAL]` badge in TUI and CLI.
- **Config merge** — `config.Merge()` combines global and project configs. `LoadGlobal`/`SaveGlobal` helpers.
- **`syllago init`** — Creates the global content dir and config during initialization; interactive provider selection and registry step.

#### TUI v3
- **Card grid pages** — Library, Loadouts, and Registries use responsive two-column card grids with scroll support.
- **Full mouse support** — Click-to-select on cards, list items, tabs, breadcrumbs, modal buttons, form fields, checkboxes, sidebar items. Click-away dismisses modals. Scroll wheel on all scrollable areas.
- **Toast system** — Centralized overlay for success, warning, and error messages. Error toasts support Esc-dismiss and clipboard copy. Progress toasts show animated spinners during async operations.
- **Breadcrumbs** — Clickable navigation trail on every page except the homepage.
- **Search** — `/` activates search on Homepage, Items, Library, Loadouts, Registries, and Detail pages.
- **Help overlay** — `?` opens a keyboard-shortcut reference organized by screen, with scroll support on small terminals.
- **Loadout create wizard** — Multi-step wizard to build a `loadout.yaml` from catalog content.
- **Flexoki palette** — New color scheme standardized across all pages.
- **Two-tier tab navigation** — Brackets for all hotkeys (`[1]`, `[a]`, `[n]`). Context-sensitive action buttons per group, right-aligned on the tab row.
- **Metadata panel** — Reusable bordered component (`metapanel.go`) shared by Library and every Content tab. Fixed-width fields prevent jank.
- **`q` backs out** instead of quitting unless on the landing page; `R` refreshes the catalog from disk without restarting.

#### Canonical Keys & Interchange
- **Six content types covered** — `canonical-keys.yaml` now includes skills, rules, hooks, mcp, agents, and commands under a single schema.
- **Normative spec in the YAML itself** — Confidence enum (`confirmed`/`inferred`/`unknown`), key lifecycle policy (add/rename/deprecate/split/remove), staleness rules, and public-spec inheritance requirements are written into the file's preamble.
- **Provider mappings populated** — `canonical_mappings` filled in across supported content types for every provider.
- **Hook canonical format** — Provider-neutral event names (`before_tool_execute`) and tool names (`shell`, `file_read`, `file_edit`) decouple the interchange format from any single provider. Defined in `docs/spec/hooks.md`.

#### Capmon & Capabilities
- **Provider format docs** — `docs/provider-formats/*.yaml` now covers 14 providers (amp, claude-code, cline, codex, copilot-cli, cursor, factory-droid, gemini-cli, kiro, opencode, pi, roo-code, windsurf, zed) with per-content-type sections.
- **Capability YAML pipeline** — `update-format-doc` agent keeps YAML and format docs in lockstep. Pre-parse validation catches `supported: unknown` drift before it merges.
- **Capmon onboard** — Scaffolds a new provider's format doc and capability YAML in a single command.
- **Confidence resolution sweep** — All `inferred`/`unknown` confidence values in the catalog resolved to `confirmed` where evidence existed.
- **MOAT conformance gap analysis** — Documented in `docs/moat-conformance.md` as a baseline for future conformance work.

#### Registry
- **`syllago registry create`** — Scaffolds a new registry with `registry.yaml`, `.gitignore`, examples, contributing guide, and an auto `git init`.
- **Progress toast** during clone/sync/remove, replacing the old misleading "Done" message.
- **Registry integration tests** — 13 offline tests using local bare git repos, plus a GitHub-backed comprehensive test gated behind `SYLLAGO_TEST_NETWORK=1`.
- **`CacheDirOverride`** for test isolation of registry operations.

### Improvements

#### Content Model
- **Configurable provider paths** — `PathResolver` supports custom path overrides for import/export/loadouts.
- **Hook import/export** — Cross-provider compatibility overhaul. The `HookAdapter` interface plus read-back verification make round-tripping hooks deterministic.
- **Provider detail view** — `syllago info providers <slug>` drills into a specific provider, including the new extension fields (Required, ValueType, Examples).
- **Non-blocking validation warnings** — `capmon validate-format-doc` and `capmon check` now emit dedup'd warnings (allow-list checks for value types, example languages, and source sections) to stderr without failing the run. Blocking errors still fail as before.

#### TUI Consistency
- Standardized card styles via `cardNormalStyle` / `cardSelectedStyle`.
- Standardized modal width (`modalWidth = 56`) and shared `renderButtons()` helper.
- Context-specific help text (`a add skill` vs `a add registry`).
- Shared page helpers: `renderBreadcrumb()`, `cursorPrefix()`, `renderScrollUp/Down()`, `renderDescriptionBox()`.
- Comprehensive `docs/design/tui-spec.md` plus 15 enforced `.claude/rules/tui-*.md`.

#### Testing
- Golden file coverage for Library, Loadout, Registries, Import, Update, and Sandbox pages at four terminal sizes (60x20, 80x30, 120x40, 160x50).
- Registry actions test suite — 38 model-level tests for add/remove/sync state transitions.
- Escape-back behavior tests across all screens.
- Keyboard and mouse scroll tests.
- Overflow tests with 85+ items.

#### Infrastructure
- **OpenWolf integration** — Context management via `.wolf/` (`anatomy.md`, `memory.md`, `cerebrum.md`, `buglog.json`). Lets agents skip full-file reads when anatomy descriptions suffice.
- **Beads 1.0.0** — Bidirectional cross-repo Dolt sync. 69 legacy `.handoffs/` JSONs migrated into the beads system.
- **Hygiene** — Untracked runtime `audit.jsonl`; `.beads.bak` added to `.gitignore`.

### Breaking Changes
- **Removed `Prompts` and `Apps` content types.** Content repos should migrate to `commands/` or `skills/`.
- **Removed `export` command.** Use `syllago install` (into a provider), `syllago convert` (for sharing), or `syllago sync`. `syllago export` now prints a redirect and exits 1.
- **Removed `promote` command.** Use `syllago share` (local→global) or `syllago publish` (→registry). `syllago promote` now prints a redirect and exits 1.
- **CLI verb vocabulary changed.** `import` is deprecated in favor of `add`.
- **Registry names use `owner/repo` format.** `NameFromURL` returns `acme/my-tools` instead of `my-tools` for GitHub URLs.
