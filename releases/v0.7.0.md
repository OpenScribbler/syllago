## v0.7.0

### Highlights

**7-Verb CLI Redesign** — The command vocabulary is now `add`, `remove`, `install`, `uninstall`, `share`, `publish`, and `sync`. Discovery mode in `syllago add` lets you browse available content interactively. Old commands (`import`, `export`, `promote`) are deprecated with helpful redirect messages.

**Global Content Library** — Content can now live in `~/.syllago/content/` alongside project-local content. Global items show a `[GLOBAL]` badge in the TUI and CLI. Config merges global + project settings automatically.

**TUI Overhaul** — Card grid pages for Library, Loadouts, and Registries. Full mouse support (click-to-select, click-away modals, scroll wheel). Toast notifications with animated spinners for async operations. Breadcrumbs, search, and Tab focus on every page. Help overlay covers all screens.

---

### New Features

#### CLI Commands
- **`syllago add`**: Discovery mode with positional args, item-level discovery, and provider detection flow.
- **`syllago remove`**: Remove content from project or global library.
- **`syllago install` / `uninstall`**: Install or uninstall content to/from providers.
- **`syllago share`**: Share content from project to global library.
- **`syllago publish`**: Publish content to a registry.
- **`syllago convert`**: Convert content between providers.
- **Deprecated commands**: `import`, `export`, `promote` redirect to their replacements with guidance.

#### Global Content
- **Global library**: `~/.syllago/content/` stores content shared across projects.
- **Config merge**: `config.Merge()` combines global + project configs. `LoadGlobal`/`SaveGlobal` helpers.
- **`[GLOBAL]` badge**: TUI and CLI mark global-sourced items.
- **`syllago init`**: Creates global content dir and config during initialization.

#### Registry
- **`syllago registry create`**: Scaffold a new registry with `registry.yaml`, `.gitignore`, examples, contributing guide, and auto `git init`.
- **Progress toast**: Animated spinner during clone/sync/remove operations instead of misleading "Done" message.
- **Registry integration tests**: 13 offline tests using local bare git repos, plus GitHub-backed comprehensive test.
- **`CacheDirOverride`**: Test isolation for registry operations.

#### TUI
- **Card grid pages**: Library, Loadouts, and Registries use card grids with two-column layout, responsive sizing, and scroll support.
- **Mouse support**: Click-to-select on cards, list items, tabs, breadcrumbs, modal buttons, form fields, checkboxes, and sidebar items. Click-away dismisses modals. Scroll wheel on all scrollable areas.
- **Toast system**: Centralized overlay for success/error messages. Error toasts support Esc dismiss and clipboard copy. Progress toasts show animated spinner with accent border.
- **Breadcrumbs**: Clickable navigation trail on all pages except Homepage.
- **Search**: `/` activates search on Homepage, Items, Library, Loadouts, Registries, and Detail.
- **Tab focus**: Tab toggles sidebar/content focus on all card pages.
- **Help overlay**: `?` shows full keyboard shortcuts organized by screen type, with scroll support on small terminals.
- **Loadout create wizard**: Multi-step TUI wizard to build loadout.yaml from catalog content.
- **Modal input navigation**: Tab cycles between form fields and buttons. Arrow keys work inside text inputs.
- **Shared page helpers**: `renderBreadcrumb()`, `cursorPrefix()`, `renderScrollUp/Down()`, `renderDescriptionBox()`.

#### Init Wizard
- **Interactive provider selection**: Choose which providers to configure during `syllago init`.
- **Registry step**: Add registries during initialization.

### Improvements

#### Content Model
- **Configurable provider paths**: `PathResolver` supports custom path overrides for import/export/loadouts.
- **Hook import/export**: Cross-provider compatibility overhaul for hooks.

#### TUI Consistency
- **Standardized card styles**: All pages use `cardNormalStyle`/`cardSelectedStyle` from `styles.go`.
- **Standardized modal width**: All modals use `modalWidth = 56`.
- **Standardized buttons**: All modals use `renderButtons()` instead of inline help text.
- **Text truncation**: Shared `truncateString()` helper used consistently.
- **Context-specific help text**: Action keys show what they do in context (e.g., "a add skill" vs "a add registry").
- **Design spec**: Comprehensive `docs/design/tui-spec.md` and 15 enforced `.claude/rules/tui-*.md` files.

#### Testing
- **Golden file coverage**: Library, Loadout, Registries, Import, Update, Sandbox pages at 4 sizes (60x20, 80x30, 120x40, 160x50).
- **Overflow tests**: Large dataset (85+ items) at multiple sizes.
- **Registry actions test suite**: 38 model-level tests for add/remove/sync UI state transitions.
- **Escape-back behavior tests**: Verify Esc navigates back on all screens.
- **Scroll tests**: Keyboard and mouse scroll on cards, items, sidebar.

### Bug Fixes

- Fixed registry showing 0 items after add (stale sources, wrong scan ordering, config path mismatch).
- Fixed modal text inputs not allowing cursor movement with arrow keys.
- Fixed CLI registry commands using wrong config location when `content_root` is set.
- Fixed scan warnings writing to stderr instead of being collected.
- Fixed Codex agent TOML and Kiro agent JSON round-trip conversion.
- Fixed loadout create scroll causing slice bounds panic.
- Fixed security notice mentioning "prompts" instead of "commands".
- Fixed orphaned clone directory left behind when config save fails after registry add.

### Breaking Changes

- **Removed `Prompts` and `Apps` content types.** These were unused and added noise. Content repos with `prompts/` or `apps/` directories should migrate to `commands/` or `skills/`.
- **Removed `export` command.** Use `syllago sync` instead.
- **Removed `promote` command.** Use `syllago share` (local to global) or `syllago publish` (to registry).
- **CLI verb vocabulary changed.** `import` is deprecated in favor of `add`. All old commands print redirect messages.
- **Registry names use `owner/repo` format.** `NameFromURL` now returns `acme/my-tools` instead of `my-tools` for GitHub URLs.
