## v0.10.0

### Highlights

**MOAT trust model shipped end-to-end** — Syllago is now the reference implementation of [MOAT](https://github.com/OpenScribbler/moat). Registries can publish signed content; `syllago` cryptographically verifies every install against Sigstore signatures, Rekor inclusion proofs, and pinned GitHub OIDC numeric IDs. Three trust tiers (`DUAL-ATTESTED`, `SIGNED`, `UNSIGNED`) surface in the TUI on Library and Registries pages, and `--min-trust` enforces a per-call policy floor.

**Monolithic rule files now install correctly** — `syllago add --from <path> --split` detects monolithic rule files (`CLAUDE.md`, `.cursorrules`, `.windsurfrules`, `.clinerules`, etc.), splits them into per-rule items on H2/H3/H4 boundaries, and installs them with full provenance history. A new `--method=append` install path writes back to the original monolithic file when that's what the provider expects.

**Install verification with state-aware updates** — `syllago install` now detects whether each target file is `Clean`, `Modified`, `Edited`, or `Missing` before writing, and routes through the right modal (or `--on-clean` / `--on-modified` flag) so updates never silently overwrite user edits.

### New Features

#### MOAT verification pipeline
- **Sigstore + Rekor verification** — Every MOAT-signed registry and item is verified against the bundled Sigstore trusted root. Rekor inclusion proofs confirm the entry was logged at the time of signing. Per-item attestations and registry signing profiles are checked independently.
- **GitHub OIDC numeric-ID pinning** — `--signing-identity`, `--signing-issuer`, `--signing-repository-id`, and `--signing-repository-owner-id` on `syllago registry add` pin the certificate's workflow SAN, issuer, and immutable repository numeric IDs. Numeric pinning defeats repo-rename and ownership-transfer attacks.
- **Bundled allowlist** — `cli/internal/moat/signing_identities.json` ships with a pre-pinned identity for `OpenScribbler/syllago-meta-registry` so the official meta-registry verifies out of the box.
- **Auto-detect from `manifest_uri`** — Registries that self-declare their MOAT manifest URI in `registry.yaml` are recognized as MOAT-capable on add.
- **Three trust tiers** — `DUAL-ATTESTED` (per-item attestation + registry profile both verify), `SIGNED` (manifest signature only), `UNSIGNED` (git transport only). `--min-trust` blocks anything below the configured floor.
- **Lockfile + revocation** — `syllago.lock` records the verified hash and trust tier per installed item. Revocations (reasons: `malicious`, `compromised`, `deprecated`, `policy_violation`) come from both the bundled archival list and live registry sources; blocked installs surface `MOAT_008`.
- **Trusted-root staleness cliff** — The bundled Sigstore root has a 90 / 180 / 365-day staleness gradient; expired roots block verification with `MOAT_005` until the binary is updated or `--trusted-root` is supplied.
- **`syllago moat sign`** — Self-publishing helper that signs a local registry's `moat-registry.json` and emits per-item attestations.
- **TUI trust surfacing** — Trust glyphs on every Library row and Registry card; a unified Trust Inspector modal (`[t]`) on both pages drills into signing identity, Rekor index, and revocation status.
- **Structured error codes** — `MOAT_001` through `MOAT_009` cover identity-pinning failures, signature failures, staleness, revocation, and policy-floor refusals. Every code is explainable at runtime via `syllago explain MOAT_NNN`.

#### Rules splitter + monolithic-file install
- **`syllago add --from <path> --split`** — Discovers monolithic rule files in a project and splits them into per-rule items on H2/H3/H4 boundaries with a literal-marker heuristic (e.g. `--- Rule: <name> ---`). Decorative HRs, code-block content, MUST/SHOULD/MAY casing, and `@import` lines are preserved.
- **TUI discovery flow** — The add wizard's Provider source now auto-detects splittable monolithic files (`CLAUDE.md`, `.cursorrules`, `.windsurfrules`, `.clinerules`, etc.), shows a heuristic radio step, and renders a multi-select review with rename overrides per rule.
- **`syllago install --method=append`** — Writes a discovered rule back into the provider's monolithic file in place rather than dropping a separate per-rule file. Empty-file create, missing-trailing-newline repair, and end-of-file dedup all handled.
- **Rulestore with version history** — Each installed rule keeps a `.source/` capture of its origin and a `.history/` chain of normalized hashes. Loading a rule with an orphan history file is a hard error.
- **Canonical normalization** — A new normalization helper (line endings, trailing whitespace, BOM stripping, etc.) ensures the same rule body hashes identically across providers and editors.

#### Install verification
- **State scanner** — `installcheck` examines installed files and classifies each target as `Clean`, `Modified`, `Edited`, or `Missing`. The library Installed column and metapanel show per-target status.
- **`--on-clean` / `--on-modified` flags** — On `syllago install`, choose how each state is resolved without entering the TUI (`overwrite`, `skip`, `keep-modified`, etc.).
- **Per-state TUI modals** — `installUpdateModal` for clean upgrades, `installModifiedModal` for user-edited targets with reason-varying defaults so the safest action is preselected.
- **`R` rescan** — Pressing rescan in the TUI re-runs the verification scan against current disk state.

#### Provider drift detection
- **`provmon` source-hash drift** — Tracks each provider's upstream config-key surface and detects when a provider changes its schema. `--fail-on=warn|error` for CI gates.
- **`syllago capmon backfill`** — Backfills capability data for providers that hadn't been examined in this cycle so the capability matrix stays current.
- **Coverage drift test** — `TestCoverageNoDrift` (gated by `SYLLAGO_COVERAGE_STRICT=1`) asserts that Go provider code, `docs/provider-formats/*.yaml`, and `docs/provider-sources/*.yaml` agree on which content types each provider supports. Drift fails the build.

#### TUI Config group
- New top-level `Config` tab with three sub-tabs: **Settings** (config file values), **Sandbox** (effective bubblewrap profile), **System** (detected providers, OS, paths). Full mouse parity across every interactive element.

#### Other features
- **Multi-provider loadouts** — `syllago-starter` ships emitters for Claude Code, Gemini CLI, Codex, and Pi from a single manifest.
- **`syllago init` interactive provider selection** — Pick which detected providers to wire up at first run.
- **Hook script scanner chain** — `syllago install` for hooks runs through a script-level security scanner with an external-subprocess adapter (Semgrep, ShellCheck) interface.
- **Telemetry: verification + decision events** — New properties for install verification state and the decision action chosen.

### Security

- **Snapshot symlink TOCTOU + backup hash verification** — Snapshot restore now verifies backup file hashes and resolves symlink targets before writing, preventing time-of-check-to-time-of-use races and tampered restores.
- **`go-tuf/v2` pinned to v2.4.1** — Patches three CVEs in the sigstore-go init chain.
- **Race-detector clean** — Two data races in tests caught by `go test -race` were fixed (no production code paths affected).

### Breaking Changes

- **Cursor commands removed** — The Cursor provider no longer advertises support for the `commands` content type. Cursor 1.6 added slash commands, but the runtime contract didn't match what syllago was emitting; shipping the broken plumbing was worse than removing it. Migration: install Cursor commands manually until upstream lands a stable contract.
