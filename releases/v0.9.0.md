## v0.9.0

### Highlights

**Five new content-type capabilities across four providers** — Roo Code now installs `.roo/commands/*.md` slash commands. Amp hooks (wired into `.amp/settings.json`) install and convert. Cline installs both workflows (as commands at `.clinerules/workflows/*.md`) and skills (`.cline/skills/<slug>/SKILL.md`). Windsurf installs Cascade Workflows as commands (`.windsurf/workflows/*.md`).

**Cursor, Zed, OpenCode, and Crush capability data filled in** — The `provider-formats/` YAMLs for these four providers were skills-only stubs in prior releases. Each now has full capability data for every content type the provider actually supports, so cross-provider conversion and `syllago info` surface accurate information.

**New coverage-drift test** — A test-time harness (`SYLLAGO_COVERAGE_STRICT=1 go test ./internal/provider/ -run TestCoverageNoDrift`) now asserts that provider Go code, `docs/provider-formats/*.yaml`, and `docs/provider-sources/*.yaml` agree on which content types each provider supports. Drift between these three sources of truth fails the build.

### New Features

#### Roo Code commands
- Added `commands` to Roo Code's supported content types (`SupportsType`, `InstallDir` at `.roo/commands`, markdown format, symlink install).
- Discovery paths: `{project}/.roo/commands` and `~/.roo/commands`.
- Matches Roo Code 3.25+ behavior where `run_slash_command` reads from `.roo/commands/*.md`.

#### Amp hooks
- Added `hooks` support to Amp. Hooks install via JSON merge into `.amp/settings.json` under the `amp.hooks` key (corrected from the earlier stale path `.amp/hooks.json`).
- Hook type: `command` (shell commands; no TypeScript/Python handlers).

#### Cline skills
- Added `skills` support to Cline. Skills install to `~/.cline/skills/<slug>/SKILL.md` with cross-provider discovery at `.cline/skills/`, `.clinerules/skills/`, and `.claude/skills/`.
- Frontmatter fields: `name`, `description` (cross-provider Agent Skills convention).

#### Cline commands (Workflows)
- Added `commands` support to Cline. Cline Workflows are first-class slash commands (`/workflow-name`) invoked in the plan-and-act loop; syllago maps them onto the Commands content type.
- Install path: `.clinerules/workflows/` (project) and `~/Documents/Cline/Workflows/` (global).

#### Windsurf commands (Cascade Workflows)
- Added `commands` support to Windsurf. Cascade Workflows are invoked via `/<workflow-name>`; syllago maps them onto the Commands content type.
- Install path: `.windsurf/workflows/` (project) and `~/.codeium/windsurf/global_workflows/` (global).

### Improvements

- **Cursor capability data** — `docs/provider-formats/cursor.yaml` now covers all 6 supported content types: rules, skills, agents, commands, hooks, mcp. Previously only skills had data; the rest were `status: unsupported` stubs despite Cursor actually supporting them (2.4 added custom subagents + skills, 1.6 added custom slash commands).

- **Zed capability data** — `docs/provider-formats/zed.yaml` now covers Zed's rules and MCP support. Zed does not support user-authorable skills, agents, commands, or hooks (built-in profiles `write`/`ask`/`minimal` only), so those are explicit `status: unsupported` entries rather than missing ones.

- **OpenCode capability data** — `docs/provider-formats/opencode.yaml` now covers rules, agents, commands, and mcp (skills coverage already existed). Agent frontmatter matches the active `sst/opencode` Zod schema (`model`, `variant`, `temperature`, `top_p`, `prompt`, `disable`, `description`, `mode`, `hidden`, `options`, `color`, `steps`, `permission`; `name` is filename-derived).

- **Crush capability data** — New `docs/provider-formats/crush.yaml` with rules, skills, and mcp coverage. Crush does not support hooks, user-authored agents, or commands upstream.

- **Amp hooks frontmatter field corrected** — The Amp hooks config location in `docs/provider-formats/amp.yaml` was pointing at `.amp/hooks.json`; the actual upstream schema puts hooks under `amp.hooks` inside `.amp/settings.json`. Corrected.

- **Source manifests reconciled** — `docs/provider-sources/*.yaml` for cursor, gemini-cli, opencode, roo-code, windsurf, zed, cline, and crush now reflect verified runtime support.

- **Coverage-drift test** — New `CheckCoverage()` library function and companion test in `cli/internal/provider/` assert agreement across Go code, format YAMLs, and source manifests. Gated by `SYLLAGO_COVERAGE_STRICT=1` for the full cross-source assertion set; the baseline drift test runs in every build.

### Bug Fixes

- **Amp hooks file format** — `Amp.FileFormat(catalog.Hooks)` was returning `FormatMarkdown` (the default) instead of `FormatJSON`. Hooks install via JSON merge, not markdown files. Fixed; `providers.json` now reports `fileFormat: json` for Amp hooks.
