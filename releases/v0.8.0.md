## v0.8.0

### Highlights

**Three new install-capable providers** — Factory Droid, Pi, and Crush now have full Go provider support. Content install, uninstall, discovery, and cross-provider conversion all work end-to-end for these agents, bringing syllago's install-capable provider count to 15.

**Cleaner provider-format docs** — Native frontmatter and config keys now render in the correct column on the documentation site, and rows no longer double-render when a canonical mapping and a provider extension describe the same field.

---

### New Features

#### Factory Droid support
- Full provider wiring for all 6 content types: rules (`AGENTS.md`), skills (`~/.factory/skills/`), custom droids (`~/.factory/droids/`), commands (`~/.factory/commands/`), MCP (`.factory/mcp.json`), and hooks (`.factory/settings.json`).
- Custom Droid frontmatter recognizes Factory's categorical tool names (`filesystem`, `shell`, `search`, `browser`, `web_fetch`) rather than per-tool allowlists.
- Detection via `~/.factory/` directory or `droid` on `PATH`.

#### Pi support
- Full provider wiring for rules (`AGENTS.md`), skills (`~/.pi/agent/skills/`), hooks, and commands (`~/.pi/agent/prompts/`).
- **TypeScript hooks** — Pi hooks are TypeScript extensions dropped into `~/.pi/agent/extensions/`, not JSON-merged config. A new `FormatTypeScript` file format preserves this distinction through the install pipeline.
- Detection via `~/.pi/` directory or `pi` on `PATH`. No MCP; no user-definable agent format.

#### Crush support
- Full provider wiring for rules (`AGENTS.md` project-only), skills (`~/.config/crush/skills/`, XDG-compliant), and MCP (`crush.json` with `stdio` / `http` / `sse` transports).
- Detection via `~/.config/crush/` directory or `crush` on `PATH`. No hooks upstream (tracked in `charmbracelet/crush#2038`).

### Improvements

- **Provider-format docs metadata** — Across 11 provider-format YAMLs, `provider_field` is now populated on canonical mappings whose mechanism cites a specific native key, and `extension_id` is set where a provider extension describes the same field. The docs site renders those keys under "Fields" instead of "Other features" and dedupes rows that previously double-rendered (e.g. `disable-model-invocation` on Claude Code skills).

### Bug Fixes

- Fixed the provider smoke workflow failing on every tag push because it was installing the wrong Gemini CLI npm package (`@anthropic-ai/gemini-cli`, which returns 404). The workflow now installs the correct `@google/gemini-cli` package.
