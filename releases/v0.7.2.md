## v0.7.2

### Highlights

**Provider smoke tests run clean again** — The end-to-end integration tests that verify `syllago loadout apply` against Claude Code and Gemini CLI no longer depend on sample content that was removed earlier; they now ship with a self-contained fixture alongside the tests themselves.

---

### Bug Fixes

- **Provider smoke tests** — The smoke suite under `tests/smoke/` was still referencing a kitchen-sink loadout and sample content that were removed when example content moved out of the repo. Every tag push ran the workflow, every run failed. Each test now seeds its temporary `HOME` from `tests/smoke/fixtures/`, a dedicated fixture that exercises all installable content types (rules, hooks, skills, agents, MCP servers, commands) so the tests are independent of whatever content the repo happens to ship.
- **Gemini CLI commands path** — A smoke assertion expected command symlinks to land in `~/.gemini/` root; the installer has always placed them under `~/.gemini/commands/`. The assertion now matches the installer.
