## v0.5.0

### Highlights

**Sandbox Wrapper** — Run AI CLI tools in bubblewrap sandboxes with filesystem isolation, network egress filtering, environment variable stripping, and config change review. `syllago sandbox run claude-code` wraps your session; changes to provider configs are diffed and require approval before being applied back.

**Distribution Pipeline** — Pre-built binaries for 6 platforms (linux/darwin/windows x amd64/arm64), an install script (`curl | sh`), Homebrew tap, `syllago update` command, SHA-256 checksums, and cosign signing. No more `git clone` + `make build` for end users.

**Two-Phase Release Workflow** — The `/release` skill now walks through an interactive flow: analyze changes, pick release type, write notes, then ship. A safety hook blocks accidental tag creation.

---

### New Features

#### Sandbox
- **`syllago sandbox run <provider>`**: Launch a provider in a bubblewrap sandbox with read-only filesystem, writable project directory, and filtered environment.
- **`syllago sandbox check <provider>`**: Pre-flight check for bwrap, socat, and provider binary.
- **Domain allowlist management**: `allow-domain`, `deny-domain`, `domains` subcommands. Ecosystem domains (npm, PyPI, Go proxy, etc.) auto-detected from project files.
- **Port allowlist**: `allow-port`, `deny-port`, `ports` for localhost service access inside the sandbox.
- **Env var allowlist**: `allow-env`, `deny-env`, `env` for controlling which environment variables reach the sandboxed process.
- **`syllago sandbox info`**: Show effective sandbox configuration for a provider.
- **Config diff-and-approve**: Provider config files are staged before the session; changes are diffed afterward. High-risk changes (MCP servers, hooks, commands) require explicit approval. Low-risk changes auto-approve with notification.
- **TUI: Sandbox settings**: Full sandbox configuration screen in the settings sidebar.

#### Distribution
- **Install script**: `curl -fsSL .../install.sh | sh` downloads the latest release binary, verifies SHA-256, installs to `~/.local/bin`.
- **Homebrew tap**: `brew install openscribbler/tap/syllago`.
- **`syllago update`**: Self-update for release builds. Downloads latest from GitHub Releases, verifies checksum, replaces binary atomically.
- **TUI update path**: The Update screen uses the same updater package for release builds, with git pull for dev builds.
- **CI pipeline**: `go test` + `go build` on every PR and push to main.
- **Release workflow**: Tag-triggered GitHub Actions workflow builds 6 binaries, generates checksums, cosign signs, creates GitHub Release, updates Homebrew formula.

#### Release Skill
- **Interactive version selection**: `/release` analyzes git history and recommends major/minor/patch with reasoning.
- **Two-phase flow**: Prepare (write notes, bump version, commit) then ship (tag, push, publish). Human approval via `AskUserQuestion` at every decision point.
- **Release guard hook**: `PreToolUse` hook blocks `git tag` and version tag pushes unless `.release-pending.yml` exists with `status: prepared`.

#### Testing
- **Golden file visual regression**: Component-level and full-app golden tests at multiple terminal sizes (60x20, 80x30, 120x40, 160x50). Catch unintended TUI layout changes.
