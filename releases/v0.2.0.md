## v0.2.0

### Highlights

**TUI Redesign** — The terminal interface has been rebuilt with a persistent sidebar + content panel layout, modal dialogs, mouse support, and a new Nesco-branded color palette.

**Security Hardening** — 12 security fixes across input validation, file operations, and configuration handling.

**Accessibility** — Full pass replacing Unicode decorations with ASCII, adding adaptive colors for light/dark terminals, and verifying WCAG AA contrast compliance.

---

### New Features

#### TUI Redesign
- **Sidebar layout**: Persistent sidebar with category navigation replaces full-screen view switching. Content panel shows items, detail, or welcome guidance alongside the sidebar.
- **Modal dialogs**: Install, uninstall, save, promote, app script, and environment setup actions now use centered modal overlays via `bubbletea-overlay`, replacing inline confirmation flows.
- **Mouse support**: Click-to-select on sidebar items, content list items, detail tabs, and action buttons via `bubblezone`.
- **Focus management**: Tab/Shift-Tab switches focus between sidebar and content panel. Modals capture all input. Esc navigates back in two stages (content → sidebar → quit).
- **Footer bar**: Breadcrumb navigation path on the right, context-sensitive keyboard shortcuts on the left.
- **Nesco palette**: Mint green primary, viola purple accent, stone muted, with adaptive light/dark variants. All colors pass WCAG AA 4.5:1 contrast ratio.

#### TUI Polish
- **Live search**: Live-filtering search with match count preview.
- **Help overlay**: `?` toggles context-sensitive keyboard shortcut reference.
- **Loading indicators**: Spinner for update operations.
- **Navigation**: Breadcrumbs, state preservation, scroll position indicators, item counters, `ctrl+n`/`ctrl+p` next/prev, Home/End keys.
- **Settings**: Auto-save on exit, large directory warnings, path preview in file browser.

#### CLI & Scanning
- `nesco scan` — Run detectors and generate context files for configured providers.
- `nesco drift` — Compare current codebase state against stored baseline.
- `nesco baseline` — Save current state as drift detection baseline.
- `nesco init` — Initialize project with provider auto-detection.
- `nesco import` — Read existing AI tool configs into canonical model.
- `nesco parity` — Compare configs across providers and report gaps.
- `nesco config` — Manage provider selection.
- `nesco add` — Add content items to the catalog.
- `nesco info` — Show capabilities, providers, and formats.
- `--json`, `--no-color`, `--quiet`, `--verbose` global flags.
- 30+ codebase detectors (tech stack, dependencies, build commands, Go/Python/Rust/TS-specific surprises).

### Security

- Strip ANSI escape sequences from rendered external text.
- Reject `git://` and `http://` clone URLs (require HTTPS).
- Whitelist MCP config fields before writing.
- Escape `.env` values with single quotes.
- Atomic config file writes via temp+rename.
- Skip symlinks in copy operations (source and destination).
- Validate item names against sjson injection characters.
- Use `0600` permissions for home-directory config files.
- Warn and preview before executing install scripts.
- Validate VERSION file content before use in ldflags.
- Require name+type match for promoted item cleanup.

### Accessibility

- Replace all Unicode decorative symbols (`▸`, `✓`, `✗`, `→`, `▲`, `△`, `★`) with ASCII equivalents.
- Convert hardcoded colors to `AdaptiveColor` for light/dark terminal support.
- Add background highlight to selected items.
- Add `Search:` label to search input.
- Prefix messages with `Done:`/`Error:` labels.
- Bracket status tags as `[LOCAL]`, `[x]`.

### Bug Fixes

- Fixed `copyFile` double-close on success path.
- Fixed `c` key conflict in file browser by changing confirm to `d`.
- Fixed missing provider names in `nesco info` formats output.
- Fixed missing success confirmations for `nesco config add/remove`.
- Fixed raw bubbletea TTY error surfacing without user guidance.
- Fixed scan silently continuing when `config.Save` or `findProjectRoot` fails.

### Build & Infrastructure

- Auto-detect OS in Makefile (`nesco` on Linux/macOS, `nesco.exe` on Windows).
- Add `build-windows-amd64` cross-compilation target.
- Add `CGO_ENABLED=0` to all cross-compilation targets.
- Migrate `filepath.Walk` → `filepath.WalkDir`.
- Replace `os.IsNotExist` → `errors.Is(err, fs.ErrNotExist)`.
- Add `t.Parallel()` to independent tests.
- Guard `syscall.Exec` for Windows compatibility.
