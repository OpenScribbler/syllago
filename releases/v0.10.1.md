## v0.10.1

### Highlights

**Release pipeline reliability fix** — The release workflow's Homebrew tap update step now fails the workflow when its credential is missing, instead of silently skipping. The v0.10.0 release exposed the gap: an inactive Aembit access policy returned an empty token, the step swallowed the error via `continue-on-error: true`, and `brew upgrade syllago` was stranded on the previous version. v0.10.1 makes the path fail loudly and adds a manual re-run trigger so a misconfiguration can be fixed and the workflow re-driven without re-tagging.

---

### Bug Fixes

- Fixed the release workflow's Homebrew tap update silently skipping when the Aembit credential step produced an empty token. The step now fails the workflow with a structured `::error::` annotation pointing at the most likely root cause (inactive access policy or wrong `credentialType`).
- Fixed `gh release create` aborting the release workflow with `a release with the same tag name already exists` on re-runs. The Create-Release step is now idempotent: if the release exists it runs `gh release edit` + `gh release upload --clobber`; otherwise it creates the release fresh.
- Fixed the Homebrew formula update aborting with `nothing to commit` when re-run against an unchanged tag. The step now early-returns when the formula is already up to date.

### Build & Infrastructure

- Added `workflow_dispatch` to the release workflow so the pipeline can be re-driven against an existing tag with `gh workflow run release.yml --ref vX.Y.Z`. No tag amendment or new tag push required to recover from a mid-pipeline failure.
- Removed `continue-on-error: true` from the Aembit credential and Homebrew formula steps. Failures in those steps now surface as workflow failures instead of green-with-warnings.
