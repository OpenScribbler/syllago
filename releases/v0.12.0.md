## v0.12.0

### Highlights

**Library tab now shows all reachable content in one place.** Registry
clone items that haven't been added to your library yet appear inline
alongside installed content, with filter chips to narrow by state. You no
longer need to know which source to check first — browse everything and
act from the same view.

**`syllago add --install --to` chains add and install in one command.**
Pass `--install --to <provider>` to add content from a registry source and
immediately install it to the named provider, skipping the two-step
add → install flow.

**Install errors for registry-only items now include the fix.** When you
try to install something that's in a cloned registry but not yet in your
library, the error message includes a copy-pasteable `syllago add <name>
--from <registry>` command so you know exactly what to run next.

### New Features

#### Library

- **Unified Library view.** Registry clone items appear inline alongside
  installed content. Items not yet added display in muted styling with a
  `[not in library]` indicator so their status is clear at a glance.
- **Filter chips.** `All`, `In Library`, `Not in Library`, and `Project`
  chips narrow the Library list. Active chips are highlighted; the default
  is `All` (unfiltered).
- **`syllago list --filter`.** New `--filter` flag accepts `in-library`,
  `not-in-library`, and `project` state values. The flag is repeatable;
  multiple values use OR logic (`--filter in-library --filter project`
  shows both).
- **Registry-add hint.** After a successful `syllago registry add`, a
  one-time TUI modal and CLI tip paragraph explain how to browse and
  install the new content. Dismissing the modal suppresses future
  appearances via a preference in `~/.syllago/config.json`.
- **`[a] Add` and `[i] Add + Install` in the TUI metapanel.** These
  buttons appear for registry items that aren't in your library yet,
  alongside the existing install and remove buttons for library items.

#### Add and Install

- **`syllago add --install --to <provider>`.** New flags that chain add
  and install in a single command. `--install` enables chaining;
  `--to` names the target provider (required when `--install` is set).

### Bug Fixes

- Fixed install error suggestions for registry-only items — the
  `INSTALL_002` structured error now includes an exact
  `syllago add <name> --from <registry>` suggestion rather than a generic
  hint.
- Fixed provider detection in the TUI install and add affordances —
  undetected providers no longer block the dialogs. Detection is now
  advisory; the affordances remain available regardless.
