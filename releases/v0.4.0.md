## v0.4.0

### Highlights

**Git Registry System** — Add git-based content sources from your team or organization. Registries are configured per-project in `.syllago/config.json` and cached globally, so teams sharing the config file get the same content automatically.

**Cross-Provider Converter** — Content is now automatically converted between provider formats during `syllago add` and `syllago export`. Rules, hooks, commands, skills, agents, and MCP configs transform between Claude Code, Cursor, Gemini CLI, Codex, Windsurf, and Copilot CLI formats.

---

### New Features

#### Registry System
- **`syllago registry add <url>`**: Clone a git repo as a content source. Prints a security warning on first use.
- **`syllago registry remove <name>`**: Remove a registry from config and delete the local cache.
- **`syllago registry list`**: Show registered registries with clone status and item counts.
- **`syllago registry sync [name]`**: Pull latest content from all (or one) registry.
- **`syllago registry items [name]`**: List items from registries with `--type` and `--json` support.
- **TUI: Registries screen**: Browse registries from the sidebar, see item counts and sync status, drill into a registry's items.
- **TUI: Landing page card**: Registries card in the Configuration section links to the new screen.
- **TUI: Registry tags**: Items from registries show `[registry-name]` prefix in item lists and detail breadcrumbs.
- **Auto-sync on launch**: Registries pull on TUI startup (5s timeout). Toggle in Settings.
- **Per-project config**: Registries stored in `.syllago/config.json` — share via git for team-wide distribution.

#### Cross-Provider Converter
- **Canonical format**: `syllago add --provider <slug>` canonicalizes content on ingest, preserving the original in `.source/`.
- **Render on export**: `syllago export --to <slug>` converts canonical format to the target provider's native format.
- **Content types**: Rules (MDC ↔ MD ↔ TOML), hooks (JSON settings), commands (MD ↔ TOML), skills (MD), agents (MD ↔ TOML), MCP (JSON configs).
- **Hooks LLM mode**: `--llm-hooks skip|generate` controls how LLM-evaluated hooks are handled during export.
- **Tool mapping**: Provider-specific tool names are mapped between providers (e.g. Bash ↔ shell, Read ↔ read_file).

#### CLI
- **`syllago add <source-path>`**: Import content from the local filesystem with automatic canonicalization, metadata generation, and README scaffolding.
- **Copilot CLI provider**: Added as a supported provider for rules and commands.
