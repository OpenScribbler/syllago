## v0.3.1

### Highlights

**Release Notes in TUI** — The Update screen now shows human-written release notes rendered with glamour instead of raw `git log` and `git diff --stat` output. When an update is available, all intermediate version notes are fetched from `origin/main` and displayed newest-first.

**View Release Notes on Latest** — When already on the latest version, a new "View release notes" option shows the current version's changelog.

---

### New Features

#### TUI
- **Glamour-rendered release notes**: "See what's new" fetches markdown release notes from `origin/main` via `git show`, filters to versions between local and remote, and renders with glamour.
- **"View release notes" option**: When on the latest version, the Update screen now shows two options — "View release notes" and "Check for updates" — instead of just "Check for updates".
- **Mouse wheel scrolling**: Release notes preview supports mouse wheel scrolling.
- **Version range header**: Preview header shows the version range (e.g. "What's new (v0.2.0 -> v0.3.0)").

### Improvements

#### TUI
- **Graceful fallback**: When no release notes files exist for the version range, falls back to the previous git log + diff stat display.
- **Enter guard**: Pressing Enter on the preview when viewing local release notes (no update available) is a no-op instead of triggering a pull.

### Internal

- **`versionInRange()` helper**: Reuses `versionNewer()` to check `localV < v <= remoteV`.
- **Moved release notes**: `docs/release-notes-*.md` relocated to `releases/v*.md` for cleaner discovery by the TUI.
