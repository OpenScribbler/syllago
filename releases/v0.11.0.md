## v0.11.0

### Highlights

**Telemetry is now opt-in.** Nothing is sent until you explicitly answer
Yes or No to a one-time consent prompt — a CLI question on the first
non-quiet command, or a modal on the first TUI launch. Existing installs
that previously inherited the opt-out default are reset and asked to
choose again; existing explicit opt-outs are preserved without
re-prompting.

**Self-update now verifies release signatures.** Release builds ship
with `checksums.txt.sig`, an Ed25519 signature over the checksum
manifest. `syllago _update` validates the signature against a public key
embedded in the binary before applying an update. Builds without an
embedded key (local development) skip verification, matching the
existing self-update tolerance.

---

### New Features

- **Telemetry consent flow.** CLI prompt for interactive sessions and a
  TUI consent modal for the App. A new `consentRecorded` flag in user
  config gates emission separately from `enabled`, so a future reset can
  re-prompt cleanly without losing the explicit-off state.
- **Build-time signer (`cli/cmd/syllago-sign`).** Stdlib-only Ed25519
  signer used by the release workflow to produce `checksums.txt.sig`.
  No third-party signing dependencies.

### Security

- **Signed release checksums.** A `checksums.txt.sig` asset is published
  alongside every release and verified by the existing self-update
  path.
- **Manual approval required to publish a release.** The release
  workflow now runs in a protected GitHub environment that gates
  binary publication on a human approval.
- **Workflow permissions scoped per job.** Removed broad top-level
  permission grants; each job declares only the scopes it needs.
- **`govulncheck` pinned.** The Go vulnerability scanner is now pinned
  to a specific version rather than tracking latest.

### Bug Fixes

- **`install.sh` line endings.** The install script could be committed
  with CRLF on some systems, which broke `bash install.sh` on Linux and
  macOS. The file is now LF-only and enforced via `.gitattributes`.
- **`syllago capmon` data accuracy.** Capability and format data was
  refreshed for Copilot CLI, Cursor, Factory Droid, and Gemini CLI;
  format coverage added for Amp, Cline, Kiro, and Pi. A new
  `gemini-cli-agents` capability file ships with the release. Source
  data and capability data are now checked for consistency at build
  time.

### Build & Infrastructure

- Scorecard workflow now exposes `workflow_dispatch`.
- Claude CI cancels superseded runs on rapid pushes.
