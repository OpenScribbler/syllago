## v0.6.0

### Highlights

**Loadouts** — Session config packaging for syllago. Bundle rules, hooks, skills, agents, MCP servers, and more into a single `loadout.yaml` that can be applied, tried temporarily, or removed with one command. `syllago loadout apply --try` sets up your session and auto-reverts when it ends.

**Content Model Restructure** — Unified content type system with 9 types (rules, hooks, skills, agents, MCP, commands, prompts, apps, loadouts), meta-content support (syllago-guide, syllago-author), and built-in TUI badge for bundled content.

**5 New Providers** — Zed, Cline, Roo Code, OpenCode, and Kiro join the supported provider list, bringing the total to 11.

---

### New Features

#### Loadouts
- **`syllago loadout apply <name>`**: Resolve, validate, and apply a loadout. Supports `--preview` (dry run), `--try` (temporary with auto-revert), and `--keep` (permanent) modes.
- **`syllago loadout remove`**: Revert a loadout using snapshot-based restore. `--auto` flag for programmatic use by SessionEnd hooks.
- **`syllago loadout list`**: List available loadouts from the catalog.
- **`syllago loadout status`**: Show currently active loadout with applied items and stale snapshot warnings.
- **`syllago loadout create`**: Interactive wizard to generate a new `loadout.yaml` from catalog content.
- **Snapshot system**: All-or-nothing apply/revert via `.syllago/snapshots/` with file backups, symlink records, and manifest tracking.
- **Stale detection**: Warns when a `--try` snapshot is older than 24 hours across all loadout commands.
- **TUI integration**: Loadout detail view with Contents and Apply tabs, mode selector, and apply modal overlay.

#### Installer Refactoring
- **`installed.json` tracking**: Replaced embedded `_syllago` JSON markers in provider config files with centralized `.syllago/installed.json`. Hooks, MCP servers, and symlinks are now tracked without polluting provider configs.
- **Clean uninstall**: Removal uses installed.json records for precise targeting instead of scanning for markers.

#### Content Model
- **9th content type**: Loadouts added to `AllContentTypes()`, catalog scanner, provider support, and TUI sidebar.
- **Meta-content**: `syllago-guide` and `syllago-author` items for self-documentation.
- **Built-in badge**: TUI displays a badge for content bundled with syllago.

#### Providers
- **Zed**: Zed editor support with rules, prompts, and commands.
- **Cline**: Cline (VS Code extension) support with rules, MCP, and commands.
- **Roo Code**: Roo Code support with rules and MCP.
- **OpenCode**: OpenCode CLI support with rules and agents.
- **Kiro**: Kiro support with rules and hooks.
