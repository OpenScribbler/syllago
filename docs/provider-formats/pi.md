# Pi — Provider Format: Skills

**Vendor:** Mario Zechner (`badlogic/pi-mono`)
**Provider version at inspection:** 0.64.0
**Standard:** Pi implements the [Agent Skills open standard](https://agentskills.io/specification)

---

## Overview

Pi uses the Agent Skills standard (agentskills.io) for its skill system. Skills are
self-contained capability packages stored as `SKILL.md` files inside named directories.
Pi scans these directories at startup, extracts names and descriptions, and exposes
them to the model via XML in the system prompt.

---

## Skill File Format

Skills are stored as markdown files with YAML frontmatter. The canonical filename is
`SKILL.md` (fixed). Each skill lives in a directory whose name must match the `name`
frontmatter field.

```
.pi/skills/
  my-skill/
    SKILL.md          ← canonical filename, required
    helper.sh         ← optional supporting scripts/assets
```

### Frontmatter Fields

| Field | Required | Constraints | Description |
|-------|----------|-------------|-------------|
| `name` | Yes | Max 64 chars. Lowercase a-z, 0-9, hyphens. Must match parent directory name. | Human-readable identifier used for invocation routing. |
| `description` | Yes | Max 1024 chars. | What the skill does and when to use it. Included in system prompt XML for model routing. |
| `license` | No | — | License name or reference to a bundled file. |
| `compatibility` | No | Max 500 chars. | Environment/platform requirements. |
| `metadata` | No | — | Arbitrary key-value mapping. |
| `allowed-tools` | No | Space-delimited list. | Pre-approved tools the skill may use (experimental). |
| `disable-model-invocation` | No | Boolean. | When true, skill is hidden from the system prompt; users must invoke via `/skill:name`. |

### Name Rules
- 1–64 characters
- Lowercase letters, numbers, hyphens only
- No leading or trailing hyphens
- No consecutive hyphens
- Must match the parent directory name (validation error if mismatched)

## Skill Locations

Pi loads skills from multiple scopes:

### Global scope
- `~/.pi/agent/skills/` — Pi-specific global skills
- `~/.agents/skills/` — Cross-provider standard global path

### Project scope
- `.pi/skills/` — Pi-specific project skills (root `.md` files also discovered here)
- `.agents/skills/` — Cross-provider standard project path (traversed up to git root or fs root)

### Additional sources
- **Packages:** `skills/` directories or `pi.skills` entries in `package.json`
- **Settings:** `skills` array in `.pi/settings.json` (files or directories)
- **CLI:** `--skill <path>` (repeatable; additive even with `--no-skills`)

### Discovery rules
- In `~/.pi/agent/skills/` and `.pi/skills/`: root `.md` files are discovered as individual skills
- In all locations: directories containing `SKILL.md` are discovered recursively
- In `~/.agents/skills/` and project `.agents/skills/`: root `.md` files are ignored (only `SKILL.md`-in-directory form works)
- Discovery can be disabled with `--no-skills`; explicit `--skill <path>` still loads

## Cross-Provider Compatibility

Pi explicitly supports reading skills from other providers' directories via settings:

```json
{
  "skills": [
    "~/.claude/skills",
    "~/.codex/skills"
  ]
}
```

For project-level Claude Code skills, add to `.pi/settings.json`:
```json
{ "skills": ["../.claude/skills"] }
```

## How Skills Work

1. At startup, Pi scans all skill locations and extracts `name` and `description` from frontmatter.
2. The system prompt includes available skills in XML format per the agentskills.io specification.
3. When a task matches a skill, the agent uses `read` to load the full `SKILL.md`.
4. The agent follows instructions, using relative paths to reference scripts and assets.
5. Users can force skill invocation via `/skill:name`.
